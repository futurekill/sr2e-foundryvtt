# Core Rules Audit — July 2026 (pre-campaign stability pass)

Systematic verification of the sr2e implementation against the SR2 core
rulebook (FASA7901 PDF, book page cited; render pages when the text layer
garbles). Goal: minimal bugs before character creation on **2026-07-10**.

Legend: ✅ verified correct · 🔧 fixed this pass · 📝 documented simplification
· ⬜ not yet audited

## 1. Ranged combat modifiers (p.88–90)
- ✅ Base TN 4 at short range; range mods +2/+4/+6 (p.88, Weapon Range Table).
- ✅ TN floor of 2 (p.88) — `Math.max(2, …)` everywhere.
- 🔧 **Attacker movement** (p.90): walking +1 (+2 difficult), running **+4**
  (+6 difficult). Was a flat "Running +2" — fixed in the attack dialog.
- 🔧 **Target stationary −1** (p.90): was missing; target select now
  Stationary −1 / Walking 0 / Running +2.
- 🔧 **Attacker in melee** (p.90): **+2 per opponent** engaging the shooter.
  Was a flat +3 checkbox — now a count input ×2.
- ✅ Multiple targets +2 per additional target (p.90) — melee dialog input.
- ✅ Aimed shot −1 per Simple Action (p.90) — via Other Mod (tooltip).
- ✅ Smartlink −2 / Smart Goggles −1, smartweapon required (p.90) — goggles
  added this pass; link supersedes goggles.
- 🔧 **Laser sight** −1 only ≤ 50 m and never with a smartlink bonus (p.90,
  p.240) — both gates were missing.
- ✅ Recoil: +1/round cumulative per phase, burst rounds count toward their
  own recoil, comp is cumulative and recoil-only (p.90, worked example
  p.92–93; vitest cites the Wedge example).
- 🔧 **Gyro-stabilization** (p.90): rating reduces recoil **and attacker
  movement** modifiers, cumulative with recoil comp. Was flat recoil comp 5.
- ⬜ Visibility/light modifiers table (p.89?) — manual via Other Mod today.
- ⬜ Second firearm (+2 each, negates smart/laser bonuses, shared recoil,
  p.90) — not automated; document as manual.

## 3. Skills & defaulting (p.69)
- ✅ Combat Pool = ⌊(Quickness + Intelligence + Willpower)/2⌋.
- 📝 Defaulting: flat +4 to the linked attribute (Skill Web is +2 per circle
  traced; the common attribute default is 2 circles = +4; full web-tracing
  is not modelled).
- ⬜ Concentrations/specializations rating math.

## 4. Damage & staging (p.91–94, p.111–112)
- ✅ Ammo types (p.93–94, audited 2026-07-01): Explosive +1 Power ✓, Gel −2
  Power/Stun/Impact armor ✓, Flechette armor = max(2×Impact, Ballistic) with
  +1 Damage Level when unarmored ✓, APDS ½ Ballistic ✓ (all in the ammo pack
  + `rollDamageResistance` armorCalc switch).
- 📝 **Knockdown (p.91) NOT implemented**: on being struck, Body Test vs TN
  ⌊Power/2⌋ (gel rounds: full Power); threshold = half wound level (L 1/M 2/
  S 3); 0 successes = prone, under threshold = 1 m stagger; Deadly always
  drops. Handle manually — candidate for an "apply damage" card button.
- 📝 Explosive-round misfire on all-ones (p.93): manual (rare edge case).
- ✅ Resistance TN = max(2, Power − armor); staging ⌊net/2⌋ per level (p.92,
  `netToSteps` tested); vehicle hard-target rules (p.108) cited in code.
- 🔧 **Worn armor stacking (p.242)**: was summing every equipped piece — now
  only the highest ballistic/impact counts, with **layered** pieces (helmets
  p.242, form-fitting body armor) adding on top (`wornArmorTotals`, tested).
- 🔧 **Heavy armor Combat Pool (p.84)**: partial/full heavy armor now reduces
  the Combat Pool by 1 die per point of Ballistic over Quickness
  (`heavyArmorPoolPenalty`, tested; flag + name fallback for old copies).
- ✅ Injury Modifier exemption on resistance/drain tests (p.112) — implemented
  (`options.isResistance`).

## 5. Magic (p.128–135, drain p.132)
- ✅ Combat-spell TN = target Willpower (mana) / Body (physical), fixed Force,
  AOE radius = Magic in metres (recent pass, p.130).
- ⬜ Drain: TN ⌊Force/2⌋ + code modifiers; Willpower resists; physical drain
  when Force > Magic.
- ✅ Sustain +2 TN per spell (spell locks exempt).
